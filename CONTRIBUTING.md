# Contributing to Backend Roadmap

Thank you for interest in improving this roadmap. Contributions are welcome.

## What We Accept

- **Content improvements** — Clearer explanations, better examples, fixing errors
- **New sections** — Additional phases or topics that fit the progression
- **Resource recommendations** — Links to high-quality learning materials
- **Bug fixes** — Broken links, typos, incorrect information
- **Diagram improvements** — Better visualizations or new diagrams

## What We Don't Accept

- **Promotional content** — Links to paid courses, boot camps, or products
- **Tool-specific deep dives** — We stay framework/language agnostic
- **Overly prescriptive guidance** — "Always do X" when context matters
- **Unnecessary complexity** — Keep it simple and clear

## How to Contribute

1. Fork the repository
2. Create a branch: `git checkout -b improve/your-change`
3. Make your changes
4. Test your changes (open roadmap.html in a browser, verify formatting)
5. Commit with a clear message: `git commit -m "Fix: clarify database indexing explanation"`
6. Push to your fork: `git push origin improve/your-change`
7. Open a pull request with a clear description

## Pull Request Guidelines

- One change per PR (easier to review)
- Reference any related issues
- Include context: why this change improves the roadmap
- Keep PRs focused and reasonably sized

## Content Guidelines

- Write for STEM graduates, not beginners
- Assume no backend experience but solid programming fundamentals
- Use principles-based explanations, not step-by-step tutorials
- Be honest about timelines and effort required
- Acknowledge tradeoffs, not just benefits

## Questions?

Open an issue or discussion to ask questions before making large changes.

Thank you for making this roadmap better for the next person learning backend development.
