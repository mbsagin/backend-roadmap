# Backend Developer Roadmap

Complete, practical roadmap for STEM graduates to become production-ready backend engineers.

## Timeline

For a strong STEM graduate working consistently:

- **9–12 months** → Possible junior hire (early-stage startups, lenient companies)
- **12–18 months** → Realistic junior hire (standard companies)
- **3–5 years** → Typical mid-level through production experience

## What You'll Learn

15 phases covering everything from networking fundamentals to production mastery:

1. **Networking** — TCP/IP, DNS, HTTP, sockets
2. **Programming Fundamentals** — Language basics, data structures, algorithms
3. **Operating Systems** — Processes, threads, memory, I/O
4. **Concurrency & Async** — Threads, locks, async/await
5. **Relational Databases** — Schema design, SQL, indexing, transactions
6. **Git & Tooling** — Version control, linters, debuggers, formatters
7. **Backend Architecture** — Services, layers, design patterns
8. **API Design** — REST principles, HTTP contracts (+ intro to gRPC)
9. **Security** — Authentication, authorization, hashing, vulnerabilities
10. **Observability** — Logging, metrics, distributed tracing, alerting
11. **Testing** — Unit, integration, E2E, test strategies
12. **Docker + CI/CD** — Containerization, automated deployment, pipelines
13. **Scalability & Distributed Systems** — Load balancing, caching, replication, message queues
14. **Cloud Fundamentals** — AWS/GCP/Azure basics, managed services
15. **Specialization** — Deep dive into your domain (fintech, e-commerce, infrastructure, real-time)

## Key Principles

- **Principles over syntax** — Understand *why* systems are built the way they are
- **Language-agnostic** — Works for Python, Go, Rust, Java, C#, Node.js, or any backend language
- **Progressive complexity** — Each phase builds on the last
- **Production reality** — Covers what's actually needed at real jobs
- **No ceremony** — Straight content, no fluff

## How This Is Different

**Learning timelines are often too optimistic.** This roadmap is honest:

- 9–12 months gets you *possible* junior hire (if you're lucky)
- 12–18 months is more realistic for standard companies
- 3–5 years of production work gets you to mid-level

Why? Learning the material is months. *Becoming* mid-level is years of debugging production, optimizing under constraint, leading projects, and mentoring others.

## Getting Started

1. Open `roadmap.html` in your browser
2. Start with **Phase 1: Networking** (1–2 weeks)
3. Work through each phase sequentially
4. Build projects combining multiple phases
5. Read others' code
6. Break things intentionally in development, learn from failures

## For STEM Graduates

This roadmap assumes:
- You have programming basics (variables, loops, functions)
- You understand algorithms and data structures at a high level
- You're comfortable with math and systems thinking
- You have no prior backend-specific experience

## Key Insight

Learning the curriculum (9–12 months) makes you hireable.

**Becoming mid-level takes 3–5 years.** Your first job teaches what courses can't:
- Debugging production at 3am
- Optimizing under real constraints
- Leading projects with uncertainty
- Mentoring junior engineers

The roadmap prepares you for year one. Production prepares you for years two through five.
